# Generate JSON Schema

This folder contains scripts to generate JSON schemas for implemented models, and compare it with the AIoD conceptual model.

## Folder Structure

- `generate_schema.ipynb`: Script to generate JSON schema, with addition to all fields from all directly/indirectly inherited parent classes.
- `generate_schema_no_inheritance.ipynb`: Script to generate JSON schemas, with only fields from directly inherited parent classes. This is used to simplify the comparison results by only showing missing fields in the parent class and not propagating them to all child classes. This simplifies the comparison.
- `conceptual_model/`: Contains taxonomies and JSON schema of the conceptual model.
- `comparison_results/`: Contains results generated by `generate_schema.ipynb`.
- `comparison_results_noinheritance/`: Contains results generated by `generate_schema_no_inheritance.ipynb`.

## Usage

1. Open the `generate_schema.ipynb` or `generate_schema_no_inheritance.ipynb` notebook in Jupyter.
2. Run the cells sequentially to generate the JSON schemas.
3. The generated results will be saved in the `comparison_results/` or `comparison_results_noinheritance/` directory, respectively.

## Code Structure

### `generate_schema.ipynb`

- **Import Libraries**: Import necessary libraries and modules.
- **Load AIoD Conceptual Model**: Load the AIoD Conceptual model.
- **Generate JSON Schema**: Generate JSON schemas for the data models.
  - **Identify Properties**: Identify properties of all parent classes the child entity is inheriting directly/indirectly.
  - **Add Properties to Schema**: Add inherited properties to the `properties` key. Note: At this stage we do not differentiate between `direct_properties` and `inherited_properties`.
- **Preprocess the generated schema**: The generated schema is preprocessed to closely match the structure of the conceptual model. As well as `ORM`, `Base`, `Type` classes are merged with their base classes (This is because they are implementational classes and do not exist in the conceptual model).
- **Generate comparison results**: Identifies missing and extra properties and entities.
- **Save Schema**: Save the generated JSON schemas to the appropriate directory.

## Comparison Results

The comparison results are stored in the `comparison_results/` and `comparison_results_noinheritance/` directories.

### CSV Files in `comparison_results/`

The `comparison_results/` folder contains four different CSV files that provide detailed comparison results:

1. **`missing_properties.csv`**: Lists the fields for each entity that are present in the AIoD conceptual model but missing in the generated JSON schemas.
2. **`extra_properties.csv`**: Lists the fields for each entity that are present in the generated JSON schemas but not in the AIoD conceptual model.
3. **`missing_entities.csv`**: Lists the entities that are present in the AIoD conceptual model but missing in the generated JSON schemas.
4. **`extra_entities.csv`**: Lists the entities that are present in the generated JSON schemas but not in the AIoD conceptual model.
